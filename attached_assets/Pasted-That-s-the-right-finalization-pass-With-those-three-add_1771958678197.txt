Global invariants (apply across T001–T007)

Deterministic routing

If templates.output_template_id is set, generation must route through that output template config.

No title heuristics, no state branching.

Single gateway

Only DocumentRenderer may emit PDF bytes or write PDF attachments.

Routes call DocumentRenderer only.

Official overlay never uses HTML/Puppeteer

mode=official_pdf_overlay must use pdf-lib only.

Flatten is mandatory

Both form-field and coordinate strategies must flatten before save.

No runtime “optional flatten” flag.

Dual-layer banned-string detection

For official overlays, run:

text extraction scan (best-effort)

raw-bytes scan (fallback)

Fail fast if either finds banned tokens.

Renderer provenance saved on every generated record

T002 — DocumentRenderer gateway

Add

DocumentRenderer.render(templateId, inputs, userId) as the only public entrypoint for document generation.

DoD

All routes call DocumentRenderer.

No other module writes PDFs or attachments.

Implement

coordinates: draw text/checkmarks → flatten → save

Both strategies flatten, always.

pdfOverlay.ts becomes internal-only implementation detail.

Not flattening

What this achieves (in plain terms)
