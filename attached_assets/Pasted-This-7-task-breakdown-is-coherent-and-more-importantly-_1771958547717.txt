Two important implementation clarifications
1) T005 “text scan for banned strings” must use a robust extraction method

Text extraction from PDFs is notoriously inconsistent depending on fonts/encoding and whether the PDF is flattened.

To avoid flaky false negatives/positives:

Use two checks:

Structural check: verify the output is derived from the base PDF (page count match is good; add “same MediaBox sizes per page” if easy).

Content check: banned-string scan using text extraction plus a fallback scan of raw PDF bytes for obvious metadata strings.

Practical:

If text extraction returns empty (common for some PDFs), the raw-bytes scan still catches “LeaseShield / Generated / Document Version / email” leaks if they were actually embedded.

2) Flattening must be mandatory for official overlays

If you fill AcroForm fields but don’t flatten, courts and users sometimes see blank forms depending on viewer.

So add a guardrail:

If mode=official_pdf_overlay: always flatten before saving.
